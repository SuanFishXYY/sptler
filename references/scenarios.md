# 专利专属场景流程指引

> sptler 除通用决策外，内置三个专利/IP 高频场景。route_sages 自动识别场景并路由专属圣人、强制 formal 轨、产出专属交付物。
> 场景配置见 `references/routing_rules.json` §scenarios。

## 场景识别

议题命中场景关键词即识别，强制正式轨（formal），召集场景必到圣人。

## 场景一：查新检索

**触发关键词**：查新、检索、新颖性、现有技术、对比文件、prior art、新颖性检索

**必到圣人**：
- 王升（结构圣）——拆解技术方案要素、判断新颖性/创造性
- 卢若雨（位控圣）——权利要求边界比对、区别特征
- 陆一帆（数据炼金圣）——检索策略、数据库、召回评估
- 邹蕴（议长）——主持
- 顾峻峰（现场组合圣·建议）——现场技术还原

**流程要点**：
1. 王升拆解技术方案为检索要素（特征→关键词→分类号）
2. 陆一帆定检索策略与数据库，评估召回
3. 全员对照对比文件，卢若雨做逐特征边界比对
4. 王升+卢若雨出新颖性/创造性判断
5. 投票：是否具备新颖性、建议保护范围调整

**交付物**：查新检索报告 `deliverable-search-{slug}-{ts}.md`（模板八-A）

## 场景二：FTO 自由实施分析

**触发关键词**：fto、自由实施、侵权风险、规避设计、侵权分析、freedom to operate、防侵权

**必到圣人**：
- 卢若雨（位控圣）——侵权比对（全要件/等同原则）、权利要求解读
- 王升（结构圣）——规避方案的结构设计
- 陈方移（基座圣）——合规留痕、时效、升级条件
- 邹蕴（议长）——主持
- 蔡悦（跨界圣·建议）——跨领域规避思路

**流程要点**：
1. 检索目标市场有效风险专利
2. 卢若雨做全要件+等同原则侵权比对
3. 王升+蔡悦设计规避方案（改特征、换路径）
4. 陈方移定留痕与复核周期
5. 投票：风险分级 + 推荐规避方案
6. ⚠️ 明确标注"内部技术分析，非法律意见"

**交付物**：FTO 分析报告 `deliverable-fto-{slug}-{ts}.md`（模板八-B）

## 场景三：专利价值评估

**触发关键词**：价值评估、专利价值、资产评估、价值分级、高价值专利、专利评级、价值盘点

**必到圣人**：
- 范征（价值圣）——价值落点、分级、运营建议
- 陆诗杰（度量圣）——评估维度与权重、量化打分
- 王升（结构圣）——权利稳定性、边界、改进建议
- 邹蕴（议长）——主持
- 卢若雨（位控圣·建议）——权利稳定性边界复核

**流程要点**：
1. 陆诗杰定评估维度与权重（稳定性/先进性/经济/战略/可执行）
2. 全员逐维度打分（0-5）+ 依据
3. 陆诗杰算加权总分
4. 范征出价值分级（S/A/B/C）+ 价值落点
5. 王升评估结构与改进建议
6. 投票：分级 + 运营建议（维持/放弃/许可/诉讼）

**交付物**：价值评估报告 `deliverable-valuation-{slug}-{ts}.md`（模板八-C）

## 通用规则

- 三场景均走正式轨（formal），完整六阶段，加权投票。
- 必到圣人不可省略；建议圣人按议题补充。
- 交付物是 headline，结果 md 是决策记录。
- 记忆照常记录（meeting_type=regular），价值分/引用计数/双画像均生效。
- 场景识别可被用户覆盖：`--track` 显式指定则不强转 formal。

## 场景四：OA答复策略

**触发关键词**：oa答复、审查意见、答复、区别技术特征、驳回答复、审查意见通知书、如何答复

**必到圣人**：
- 卢若雨（位控圣）——区别技术特征提炼、答复口径、权利要求修改
- 王升（结构圣）——权利要求结构修改方案、修改超范围判断
- 徐奕阳（铸模圣）——高频OA答复口径铸成可复用模板
- 邹蕴（议长）+ 陈彤（流程圣·建议）——OA流程节点

**流程要点**：
1. 卢若雨解读审查意见，提炼审查员引用的对比文件与驳回理由
2. 全员找区别技术特征，判断区别是否被对比文件启示
3. 王升定权利要求修改方案（不超范围、收窄保核心）
4. 徐奕阳把答复口径铸成模板（可复用于同类OA）
5. 投票：答复策略 + 修改方案

**交付物**：OA答复策略 `deliverable-oa-response-{slug}-{ts}.md`（模板八-D）

## 场景五：专利布局选型

**触发关键词**：布局选型、申请哪个、优先申请、专利布局、选哪个方案、申请优先级、哪个申请、申请专利

**必到圣人**：
- 王升（结构圣）——各方案权利要求结构可行性
- 范征（价值圣）——各方案价值落点与战略意义
- 陆诗杰（度量圣）——多维打分排序
- 邹蕴（议长）+ 卢若雨/蔡悦（建议）

**流程要点**：
1. 陆诗杰定评估维度（新颖性/稳定性/价值/规避难度/申请成本）与权重
2. 全员对每个候选方案逐维打分
3. 陆诗杰算加权总分，出优先级排序
4. 范征定价值落点，王升定结构可行性
5. 投票：申请优先级排序 + 布局建议

**交付物**：专利布局选型报告 `deliverable-portfolio-{slug}-{ts}.md`（模板八-E）

## 场景六：无效宣告攻防

**触发关键词**：无效宣告、无效请求、无效理由、无效证据、专利无效、无效攻防、invalidation

**必到圣人**：
- 卢若雨（位控圣）——权利要求解读、无效理由（新颖性/创造性/清楚性/支持）
- 陈方移（基座圣）——证据组合、程序合规、留痕
- 王升（结构圣）——权利要求结构弱点、修改超范围
- 邹蕴（议长）+ 徐伟（建议，硬科技证据）

**流程要点**：
1. 卢若雨解读目标专利权利要求，找结构弱点
2. 全员列举无效理由（新颖性/创造性/公开不充分/权利要求不清楚/得不到支持）
3. 陈方移组合证据链（对比文件+公知常识），评估证据强度
4. 王升评估修改超范围风险（对方可能如何限缩）
5. 投票：无效理由优先级 + 证据组合 + 成功率评估

**交付物**：无效宣告攻防报告 `deliverable-invalidation-{slug}-{ts}.md`（模板八-F）
