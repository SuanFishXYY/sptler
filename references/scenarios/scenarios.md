# 专利专属场景流程指引

> sptler 除通用决策外，内置六个专利/IP 高频场景。route_sages 自动识别场景并路由专属圣人、产出专属交付物。
> **每个场景有专属Phase流程**——不是统一六阶段套所有场景。场景命中时读对应场景的Phase步骤。
> 场景配置见 `references/scenarios/routing_rules.json` §scenarios。

## 场景识别

议题命中场景关键词即识别，强制正式轨（formal），召集场景必到圣人。用户在Phase 0选"快速会议"时降级为场景快评轨（必到圣人保留，规模≤5人）。

## 场景一：查新检索 — 专属流程

**触发关键词**：查新、新颖性、现有技术、对比文件、prior art、能不能申请、可专利性、有没有被公开

**必到圣人**：王升（特征拆解）、卢若雨（边界确认）、陆一帆（语义扩展+检索式）、邹蕴（主持+完备性自检）

**专属Phase流程（按 `references/scenarios/feature_analysis.md` 框架执行）：**

### Phase S1-1：三层交叉拆解（特征拆解角色主笔）
- 权要级：逐条权要拆（独权→从属权），标检索优先级
- 子特征级：每个特征拆子特征（A→A1/A2/A3），标独立可检索+依赖
- 技术维度：每个子特征按 材料/结构/方法/参数 4维交叉拆解
- 技术问题+技术效果：三要素保留

### Phase S1-2：等同特征簇（语义扩展角色主笔）
- 每个子特征：同功能不同结构的等同方案 2-3 种
- 标竞品优先（哪种等同最可能被竞品使用）
- 分类：已知等同/潜在等同/规避设计

### Phase S1-3：双路语义扩展（语义扩展角色主笔）
- 关键词路 11 维：同义/近义/上下位/英文/多语种/分类号分层/功能表达/共现词/标准化术语/竞品惯用语/缩写变体
- 语义路 4 维：概念描述/问题-效果对/类比表达/否定边界

### Phase S1-4：检索式 + 迭代计划（语义扩展角色主笔）
- 关键词路：核心式(窄)+扩展式(宽)+全称式(最宽)+否定过滤
- 语义路：核心查询+扩展查询+否定查询
- 迭代计划：第一轮窄检(top50)→第二轮宽检(top100)→第三轮引文追踪
- 场景策略：查新=高召回，分类号用大组，否定边界放宽

### Phase S1-5：产出检索准备文档
- 按 feature_analysis.md §七 模板产出结构化 Markdown
- 9 章节：概述/权要拆解/等同簇/关键词路/语义路/检索式/策略/否定边界/自检

### Phase S1-6：检索完备性自检（议长主持）
- 16 条自检（见 feature_analysis.md §七 模板末尾）

### Phase S1-7：用户提供对比文件 → 逐子特征新颖性比对
- 用户提供对比文件清单（Phase 1数据采集）
- 逐子特征 vs 每个对比文件：是否被公开？区别子特征？
- 无对比文件：只出检索准备文档，结论待检索后复核

### Phase S1-8：创造性判断
- 区别技术特征 → 实际解决的技术问题 → 是否存在技术启示

### Phase S1-9：投票 + 结论
- 可专利性综合结论（新颖性+创造性）
- 风险预警（哪些子特征最易被无效）
- 保护范围建议（收窄/分案/维持）

**交付物**：查新检索报告 `deliverable-search-{slug}-{ts}.md`（模板八-A）

## 场景二：FTO自由实施分析 — 专属流程

**触发关键词**：fto、自由实施、侵权风险、规避设计、侵权分析、freedom to operate、防侵权

**必到圣人**：卢若雨（侵权比对）、王升（规避结构设计）、陈方移（合规留痕）、邹蕴（主持）、蔡悦（跨界规避·建议）

**专属Phase流程（按 `references/scenarios/feature_analysis.md` 框架执行）：**

### Phase S2-1：三层交叉拆解 + 等同特征簇（特征拆解角色主笔）
- 权要级：逐条解读风险专利权利要求（独权→从属权）
- 子特征级：每个必要特征拆子特征，标独立可检索+依赖
- 技术维度：材料/结构/方法/参数 4 维交叉
- 等同特征簇重点展开：标已知等同/潜在等同/规避设计（FTO 核心）

### Phase S2-2：双路语义扩展（语义扩展角色主笔）
- 关键词路 11 维 + 语义路 4 维（同查新）
- 竞品惯用语重点：用风险专利权人的措辞检索其专利组合
- 否定边界严格：排除所有非等同方案

### Phase S2-3：检索式 + 迭代计划（语义扩展角色主笔）
- 场景策略：FTO=高精度，BM25 只搜 claims 字段，分类号用小组
- 迭代：第一轮精准(必要特征精确同义词)→第二轮等同(等同方案全展开)→第三轮逐条比对确认

### Phase S2-4：产出检索准备文档
- 按 feature_analysis.md §七 模板产出

### Phase S2-5：侵权比对（边界角色主笔）
- 逐子特征全要件原则比对：产品子特征 vs 风险专利子特征
- 等同原则检查：子特征是否等同（手段-功能-效果三要素）
- 输出侵权比对表（逐子特征级，非特征级）

### Phase S2-6：规避设计（结构角色+跨界角色）
- 针对落入的风险专利设计规避方案
- 规避策略：改特征（替换/省略/逆序）、换技术路径、组合规避
- 每个方案标注：是否仍落入、可行性、成本

### Phase S2-4：合规留痕（合规角色）
- 分析留档：检索式/比对表/结论存档
- 时效：风险专利到期前每年复核
- 升级条件：风险专利转让/新专利出现/等同判定变化 → 转正式法律意见

### Phase S2-5：投票 + 结论
- FTO结论：低风险/中风险需规避/高风险
- 推荐规避方案
- ⚠️ 标注"内部技术分析，非法律意见"

**交付物**：FTO分析报告 `deliverable-fto-{slug}-{ts}.md`（模板八-B）

## 场景三：专利价值评估 — 专属流程

**触发关键词**：价值评估、专利价值、资产评估、价值分级、高价值专利、专利评级、价值盘点、值不值钱、能卖多少

**必到圣人**：范征（价值落点+分级）、陆诗杰（维度打分+权重）、王升（结构+稳定性）、邹蕴（主持）、卢若雨（稳定性复核·建议）

**专属Phase流程：**

### Phase S3-1：评估对象确认 + 维度定权重（度量角色主笔）
- 确认评估对象：单件/组合/领域
- 定5维度权重：权利稳定性/技术先进性/经济价值/战略意义/可执行性
- 权重按案件类型调整：诉讼→经济0.35；许可→经济0.30+可执行0.20；防御→稳定性0.35+战略0.25

### Phase S3-2：逐维度打分（全员）
- 每维度0-5分 + 依据（可辩护的量化理由）
- 稳定性：权要边界/无效历史/审查历史
- 先进性：相对现有技术的领先程度
- 经济价值：许可/诉讼/产品绑定/市场增长
- 战略意义：布局/卡位/标准/防御
- 可执行性：检测难度/取证难度/规避难度

### Phase S3-3：加权计算 + 分级（度量角色+价值角色）
- 加权总分 = Σ(维度得分 × 权重)
- 分级：S核心(≥4.0)/A重要(3.0-3.9)/B一般(2.0-2.9)/C边缘(<2.0)

### Phase S3-4：结构评估（结构角色）
- 保护范围宽窄、边界清晰度、改进建议（补从属/分案/放弃）

### Phase S3-5：投票 + 运营建议
- 价值分级确认
- 运营建议：维持/放弃/许可/诉讼/质押
- 监控指标 + 复评周期

**交付物**：价值评估报告 `deliverable-valuation-{slug}-{ts}.md`（模板八-C）

## 场景四：OA答复策略 — 专属流程

**触发关键词**：oa答复、审查意见答复、审查意见通知书、区别技术特征、oa response、驳回答复、如何答复审查

**必到圣人**：卢若雨（区别特征+修改方案）、王升（结构修改+超范围判断）、徐奕阳（答复口径铸模）、邹蕴（主持）、陈彤（OA流程·建议）

**专属Phase流程：**

### Phase S4-1：审查意见解读（边界角色主笔）
- 审查员引用的对比文件：D1/D2/…各公开了什么
- 驳回理由：新颖性（单篇对比）/创造性（组合对比）/其他
- 涉及权利要求：权1/权3/…

### Phase S4-2：区别技术特征提炼（边界角色+结构角色）
- 三层交叉拆解本申请+对比文件：权要级→子特征级→技术维度
- 逐子特征比对：本申请子特征 vs D1/D2/…各子特征
- 找区别子特征：哪些子特征对比文件没公开（子特征级比特征级更精细）
- 判断区别是否被启示：D1+D2有无结合动机

### Phase S4-3：权利要求修改方案（结构角色主笔）
- 修改策略：从属提升独权/补入限制/删除特征
- 不超范围依据：说明书记载位置
- 修改后保护范围评估

### Phase S4-4：答复口径铸模（铸模角色主笔）
- 区别特征 → 实际解决的技术问题 → 技术效果
- 对比文件无结合启示的理由
- 可复用于同类OA的答复口径模板

### Phase S4-5：投票 + 结论
- 答复策略：维持/修改/分案
- 预期成功率：高/中/低

**交付物**：OA答复策略 `deliverable-oa-response-{slug}-{ts}.md`（模板八-D）

## 场景五：专利布局选型 — 专属流程

**触发关键词**：布局选型、申请哪个、优先申请、专利布局、选哪个方案、申请优先级、哪个申请、申请专利、先申请哪个

**必到圣人**：王升（结构可行性）、范征（价值落点）、陆诗杰（多维打分排序）、邹蕴（主持）、卢若雨/蔡悦（建议）

**专属Phase流程：**

### Phase S5-1：候选方案梳理（结构角色+价值角色）
- 列出所有候选技术方案
- 每个方案的技术要点 + 可专利性初判

### Phase S5-2：评估维度定权重（度量角色主笔）
- 维度：新颖性/稳定性/价值/规避难度/申请成本
- 权重按目的调整：防御布局→稳定性+规避；进攻布局→价值+新颖性

### Phase S5-3：逐方案逐维度打分（全员）
- 每方案每维度0-5分 + 依据

### Phase S5-4：加权排序 + 布局建议（度量角色+价值角色）
- 加权总分排序
- 布局策略：核心专利（独权宽）+ 外围专利（从属包围）+ 防御专利（占位）

### Phase S5-5：投票 + 结论
- 申请优先级排序
- 布局建议：哪些先申请、哪些分案、哪些暂缓

**交付物**：专利布局选型报告 `deliverable-portfolio-{slug}-{ts}.md`（模板八-E）

## 场景六：无效宣告攻防 — 专属流程

**触发关键词**：无效宣告、无效请求、无效理由、无效证据、专利无效、无效攻防、invalidation、打无效

**必到圣人**：卢若雨（权利要求解读+无效理由）、陈方移（证据组合+程序合规）、王升（结构弱点+修改超范围）、邹蕴（主持）、徐伟（硬科技证据·建议）

**专属Phase流程：**

### Phase S6-1：三层交叉拆解目标专利（特征拆解角色主笔）
- 权要级：逐条解读目标专利权利要求（独权→从属权），标检索优先级
- 子特征级：每个特征拆子特征，找结构弱点（子特征模糊/缺支持/公开不充分）
- 技术维度：材料/结构/方法/参数 4 维交叉，找维度级弱点

### Phase S6-2：无效理由列举（全员）
- 新颖性：有无单篇对比文件公开全部必要特征
- 创造性：组合对比文件+公知常识能否否定
- 公开不充分：说明书是否支持权利要求
- 权利要求不清楚/得不到支持

### Phase S6-3：证据组合（合规角色主笔）
- 每条无效理由配证据链：D1+D2+公知常识
- 证据强度评估：对比文件公开日/公开内容/相关性
- 程序合规：证据形式/举证期限

### Phase S6-4：对方可能应对（结构角色）
- 对方可能限缩修改：删除特征/收窄范围
- 修改超范围风险：对方修改是否合规则

### Phase S6-5：投票 + 结论
- 无效理由优先级排序
- 推荐证据组合
- 综合成功率评估：高/中/低

**交付物**：无效宣告攻防报告 `deliverable-invalidation-{slug}-{ts}.md`（模板八-F）

## 通用规则

- 六场景默认走正式轨（formal），**各走专属Phase流程**（不是统一六阶段）；用户选"快速会议"时降级为场景快评轨。
- **能力边界**：见 scenarios.md 末尾能力边界声明（sptler不检索/不撰全文/不代写诉讼）。
- 必到圣人不固定，由 route_sages 按议题动态路由 + auto_invite 智能补人。
- 交付物是 headline，结果 md 是决策记录（专利场景瘦版）。
- 记忆照常记录（meeting_type=regular），价值分/引用计数/双画像均生效。
- 场景识别可被用户覆盖：`--track` 显式指定则不强转 formal。
