## Understanding Summary

1. **在做什么**:新增第 7 个专利场景「专利挖掘」——从不完整技术方案描述,反问补全三要素,提炼可专利点,产出权利要求骨架 + 完整说明书+实施例,收口布局 hint。
2. **为什么**:横向扩 IP 边界,补上游高频缺口。
3. **为谁**:代理师/专利工程师。
4. **关键约束**:反问 elicitation(1 AskUser,允许部分答 D39);全局"草稿OK/终稿NO";pre-filing 安全(强制 sptler-meetings/,禁直投 D30);触发碰撞 disambiguation(D40)。
5. **non-goals**:不做检索、不做创造性评估、布局不展开。
6. **能力边界**:草稿级(结构性缺失 D17 + 补全 checklist D45),不产出可提交终稿。

## Assumptions

- A1 目标用户=代理师;反问 scoped 专利域可答,不可答→resume 契约(D38/D39)
- A2 说明书 ≤2500 词(D31);≤4 turns
- A3 草稿=pre-filing,强制 sptler-meetings/(gitignored),禁直投(D30),邹蕴 announce(D41)
- A4 草稿质量,人工复核,非 filing-ready
- A5 代理师可在草稿上直接完善(含补全 checklist D45)
- A6 对齐 6 场景结构 + validator gate

## Final Design (R3)

### §1 架构

第 7 个专利场景,`flow_type`=parliament。触发:`route_sages` 关键词(挖掘/可专利点/交底书提炼/权利要求草拟/发明挖掘)。**触发碰撞(D40)**:与查新("能不能申请")/布局("申请专利")碰撞时,邹蕴 declaration"这像挖掘(出草稿),要查新/布局请明说"+进行挖掘,用户可 redirect。必到:王升/卢若雨/范征/邹蕴;徐奕阳不必到。轨道:formal 默认;lite 轻量版(D11/D15);briefing 快评+`briefing_quality_concern`。

### §2 组件

**新增**:scenarios.md §挖掘 / templates.md 模板十(⚠️草稿+结构性缺失声明 D17+**补全 checklist D45**) / routing_rules.json 挖掘(`feature_analysis:false` D36)/ validate_sptler.py 挖掘 gate

**改动**:SKILL Discipline 15(carve-out D19)/ README:129 / orglaw.md / scenarios.md:261(D27)/ sptler-lite SKILL(**line 29 更新"硬拒绝(挖掘例外:仅骨架)"D42** + lite-挖掘轻量版) / sptler-brief SKILL / **.gitignore 加 patents/**(D30) / route_sages.py(D15) / record_memory.py(D16 `scenario`+D32 `low_value`+D38 `elicitation_incomplete` flag) / compact_memories.py(D16+D29 skeleton archive) / summon_sage.py(D16+D21+D34 降权;D38 续议 incomplete NOT 降权) / routing_rules.json(D36 `feature_analysis:bool`)

**安全(D30/D41)**:挖掘 deliverables 强制 sptler-meetings/(gitignored),禁直投 patents/;邹蕴 deliver 时 announce"pre-filing 敏感,已写 sptler-meetings/ 不进 patents/"。

### §3 数据流

**Standard `/sptler` 挖掘**(≤4 turns D31):
1. 输入→`scenario=挖掘`(碰撞→D40 declaration);Phase 1 召集+memory+名单 auto-proceed
2. Phase 2 反问 elicitation(D18 动态+D39 部分答):三要素检测 rubric→缺哪个→圣人动态生成必问→邹蕴合并 1 AskUser→**用户可部分答/跳过未知**(D39,邹蕴明说"不会可跳过,基于现有挖+标 incomplete")→scoped 专利域;不可答→`elicitation_incomplete`+**resume 契约**(D38:邹蕴说"问发明人X,用`/sptler 续议`回,我接着挖")
3. Phase 3+4 组合+四律(结构律=王升主查 D20;铸模=bonus)
4. Phase 5 产出骨架+说明书+实施例+布局 hint+⚠️草稿标记+**补全 checklist D45**;5b auto-write;说明书 ≤2500 词(D31),截断→`draft_quality_concern` **USER-VISIBLE D44**(收口说"实施例N截断,建议补全");deliver 强制 sptler-meetings/(D30)+announce(D41);memory 记 `scenario=挖掘`+`low_value`(若0)+`elicitation_incomplete`(若未补全);**收口给布局方向 hint(D8),拆N个申请决策移到收口 D46**

**Lite `/sptler#` 挖掘**(D11/D15/D42):`route_sages --lite`→非 lite_rejected→≤3 人;**邹蕴 announce"lite-挖掘仅骨架无说明书,要完整用标准 /sptler"**(D42);无反问;骨架 only;lite memory

**AskUser**:1(反问,允许部分答 D39),在 scenario zero-or-one 内(D24)。

### §4 错误处理

| 失败模式 | 检测 | 响应(用户可见) |
|---|---|---|
| 反问不可答/不完整 | 邹蕴检测补全度 | `elicitation_incomplete`;**resume 契约**(D38):"问发明人X,用`/sptler 续议`回";续议时此 memory NOT 降权 |
| 触发碰撞 | 查新/布局关键词也命中 | 邹蕴 declaration(D40)+"这像挖掘,要查新/布局请明说"+进行挖掘 |
| 圣人覆盖盲区 | route 兜底 | `domain_coverage_concern`;诚实说 |
| 说明书质量不达草稿级 | 结构律检查 | 补写;`draft_quality_concern` |
| 说明书截断 | ≤2500 词 | `draft_quality_concern` **USER-VISIBLE**(D44):收口"实施例N截断,建议补全" |
| 草稿触及终稿红线 | 邹蕴自检结构性要素 | 删除降级 |
| 挖出 0 可专利点 | Phase 4 共识 | 出骨架+`low_value`+⚠️+**action(D43)**:"骨架仅结构参考,勿据撰;先查新确认无先术再补强" |
| lite-挖掘输入过缺 | 三要素检测 | `quality_concern`+"建议标准";`lite_insufficient_input` |
| briefing-挖掘输入过缺 | 三要素检测 | `briefing_insufficient_input`+`briefing_quality_concern` |

### §5 边界用例

| 边界 | 处理 |
|---|---|
| 挖掘×invites | 邀请不截断 |
| 挖掘×记忆 | `scenario=挖掘`×0.5 降权 |
| 挖掘×compact | skeleton archive(D29),不物理删 |
| 挖掘×superseded | 同圣人 only;跨圣人=降权+recency |
| 挖掘×low_value | ×0.25 multiplicative(D34) |
| 挖掘×elicitation_incomplete(D38) | **续议时**:`elicitation_incomplete` 因子 lift(NOT 降权),`scenario=挖掘` 因子保留 ×0.5 → 有效 ×0.5;**标准引用时**:×0.5(scenario)×0.5(incomplete)=×0.25 |
| 挖掘×多发明 | 收口"拆N个申请"(D46),不挤反问 |
| 挖掘×briefing | ≤5+跳反问+`briefing_quality_concern`+输入过缺走 D26 |
| 挖掘×第二次同主题 | 引用降权,不当定论 |
