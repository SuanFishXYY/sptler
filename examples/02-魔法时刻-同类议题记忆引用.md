# 示例二：记忆魔法时刻·同类议题的圣人引用

> 这是 sptler 最能体现"系统有积累"的瞬间。本例发生在示例一的 OA 流水线议会之后，用户又问了一个**同类**问题。圣人开口第一句就引用了上次经历——这是裸 Claude 做不到的。

## 背景

示例一议会结束后，徐奕阳/张鑫/王升等都记下了"OA流水线"的经历。现在用户问：

`/sptler OA答复的Prompt模具具体怎么设计？`

## Phase 1 召唤（summon_sage.py --topic 注入）

路由判定这次仍为快速轨，徐奕阳再次入会。召唤时带上了当前议题：

```bash
python scripts/summon_sage.py --sage 徐奕阳 --topic "OA答复的Prompt模具怎么设计"
```

脚本自动从徐奕阳的记忆里筛出**与当前议题最相关**的历史经历：

```
## 记忆摘要
参与 1 次；风险偏好：平衡中立；焦点：AI/铸模
常提观点：三段式Prompt模具；人机协同边界显式；评估闭环

## ⚡ 可引用的相关记忆（请在发言第一句引用）
  - [2026-06-15 14:00] OA审查意见答复AI流水线（快速会议/通过）：
    立场=赞成，建议=先做高频OA的3个种子Prompt
```

## Phase 2 发言（强制引用记忆 → 魔法时刻）

> Discipline 第13条：有相关记忆的圣人，发言第一句必须引用。

```
[徐奕阳] 上次我们议过 OA 流水线(06-15)，当时定的是"三段式Prompt模具+半自动起步"。
         这次我直接按那个骨架展开——输入段怎么设计：
  - 设想1：输入段Prompt要含【法条引用】+【审查点】+【申请人原始权利要求】三要素
  - 设想2：输出段固定"技术反驳+口径建议"双轨
  - 设想3：评估段用规则卡触发37条检查
```

这一句"上次我们议过……当时定的是……"，就是用户瞬间感到"这套系统比直接问 Claude 强"的魔法时刻——圣人记得自己的历史判断，并在新问题上接续，而不是每次从零开始。

## 对比：如果张鑫也被问到

```bash
python scripts/summon_sage.py --sage 张鑫 --topic "OA答复流水线要不要全自动"
```

```
## ⚡ 可引用的相关记忆
  - [2026-06-15] OA流水线：立场=反对，建议=先补异常分支和留痕再自动化

## 关系摘要
- 徐奕阳: 共议1次，最近：OA审查意见答复AI流水线
- 徐骋:   共议1次，最近：OA审查意见答复AI流水线
```

张鑫开口会引用自己**上次投了反对票**的原因，而不是重复一遍相同的论证。这就是记忆 + 关系网让圣人"活起来"的效果。

## 为什么这是 sptler 的核心价值

| 维度 | 裸 Claude | sptler（有记忆） |
|---|---|---|
| 同类问题第二次问 | 从零开始，重复论证 | 引用上次结论，直接接续 |
| 圣人立场一致性 | 无保证，可能前后矛盾 | 记得自己上次赞成/反对的理由 |
| 团队协作上下文 | 不知道谁和谁议过什么 | 关系网显示共议历史 |

> 续议到此结束。
