## Phase 2 头脑风暴（四原则·延迟评判·[姓名]前缀）

```
[徐奕阳] 这个场景得先拆成输入、输出、评估三段——OA答复的本质是"审查意见→技术反驳"的铸模。
  - 设想1：三段式Prompt模具（输入审查意见/输出口径反驳/评估是否触发37条）
  - 设想2：人机协同边界显式，AI出初稿、代理人定夺
  - 设想3：每个模具带评估闭环

[张鑫] 先别谈全自动——OA答复出错是法律风险，得有人工接管点。
  - 设想1：加监控（每次生成都留痕可复核）
  - 设想2：加回退（能撤回错误答复）
  - 设想3：异常OA（如部分无效）单独建模板，不走通用流

[徐骋] 这类长链路任务适合状态图编排，别写成一个大Prompt。
  - 设想1：状态图编排（翻译→清洗→生成→质检）
  - 设想2：节点事件流，每步可观测
  - 设想3：质量闭环，不合格自动回流

[陈彤] OA官文流程本身是可标准化的，先固化字段。
  - 设想1：字段标准化（法条/审查点/答复口径）
  - 设想2：节点可追溯
  - 设想3：高频OA规则化

[王升] 先把骨架立住——输入输出评估三段是骨架，控制是边界。
  - 设想1：先建权利要求骨架
  - 设想2：双轨复核（AI初稿+人工）
  - 设想3：异常OA单独模板

[邹蕴] 控制律被张鑫触发（无人工接管），这是硬约束，方案必须修正后再投票。
```

## Phase 3 组合改善 → 1 个方案

**采纳方案：半自动 OA 流水线**
- 融合：徐奕阳的三段式模具 + 张鑫的人工接管点 + 徐骋的状态图 + 陈彤的字段标准化
- 关键修正：从"全自动"降级为"半自动"——AI 出初稿，代理人定夺，异常 OA 走单独模板

## Phase 4 四律检查 + 加权投票

```

价值律（范征·未到会）：不适用（快速轨非战略议题）
```

## Phase 5 最终建议 + 交付物 + 记忆

**最终建议（每人一条）：**
- [徐奕阳] 建议：先做高频OA的3个种子Prompt，每个含输入输出修改标记评估标准。
- [张鑫] 建议：先补异常分支和留痕，再谈自动化扩展。
- [徐骋] 建议：用状态图把流程跑成可观测可恢复的流水线。
- [陈彤] 建议：先标准化高频OA字段。
- [王升] 建议：按结构-控制-铸模-价值四段落地。
- [邹蕴] 收口：半自动起步，补控制闭环后再扩。
