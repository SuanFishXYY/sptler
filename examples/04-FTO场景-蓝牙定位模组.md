```
[卢若雨] 上次我们议过蓝牙定位模组FTO(半年前)，当时定的是用到达角(AOA)替代接收信号强度(RSSI)来规避。
         这次新款模组如果还是RSSI方案，要先查那件风险专利是否仍有效。
  - 设想1：核验用户提供风险专利的有效性（年费/无效）
  - 设想2：新款若用AOA，需重新做全要件比对
  - 设想3：等同原则下AOA是否仍可能落入

  - 设想1：规避不止算法，硬件布局也要差异化
  - 设想2：权利要求拆解成"算法层+硬件层"分别比对
  - 设想3：边界：AOA方案的保护范围收窄到什么程度才安全

[陈方移] FTO分析必须留痕，这是合规底线。
  - 设想1：分析过程全留档（检索式、比对表、结论）
  - 设想2：标注"内部技术分析，非法律意见"
  - 设想3：定复核周期——风险专利到期前每年复核

[张鑫] 控制律检查：FTO结论若被用于上市决策，必须有"结论失效"的兜底——比如风险专利转让/新专利出现。
  - 设想1：设风险监控触发器
  - 设想2：高风险时必须升级正式法律意见

[蔡悦] 跨界思路：定位领域除了蓝牙，UWB方案有没有绕开蓝牙专利池的可能？
  - 设想1：评估UWB替代的技术可行性
  - 设想2：双模方案分摊风险

[张鑫] 蔡悦这个双模我持保留——双模不是分摊风险，是扩大侵权比对面。蓝牙+UWB两套都查，工作量翻倍且漏检风险更大。我的本能是先单模AOA跑通，UWB作为纯备选不进产品。

[邹蕴] 提醒：陈方移的留痕和张鑫的兜底都触及控制律，投票前要确认FTO报告含失效触发器。
```

## Phase 5 最终建议

- [卢若雨] 建议：新款若用AOA，重点比对风险专利权1的"信号处理单元"特征。
- [王升] 建议：硬件布局差异化至少1处，避免等同原则落入。
- [陈方移] 建议：报告存档并设每年复核，风险专利到期前必复核。
- [张鑫] 建议：高风险专利转让时触发重新分析。
- [蔡悦] 建议：UWB备选方案做技术预研，分摊单一蓝牙路径风险。
- [邹蕴] 收口：议会决定采用AOA主导规避方案——新款蓝牙定位模组主用到达角(AOA)、RSSI作降级备选，不落入风险专利权1。关键理由：卢若雨引用半年前FTO经验确认AOA规避有效，王升确认硬件层也需差异化。风险：需年度复核风险专利有效性+UWB备选预研，风险专利转让时触发重新分析。

## 这个示例展示的关键能力

1. **场景识别**：FTO关键词命中 → 自动formal轨 + 必到圣人 + 交付目录
2. **主动提醒+魔法时刻**：watch_memory 提醒卢若雨有历史经验 → 召唤时返回 → 发言第一句引用
3. **四律检查**：控制律要求FTO报告含失效触发器（张鑫触发，陈方移落地）
4. **交付物直投**：报告写 patents/fto-reports/，非 sptler-meetings/ 堆
5. **加权投票**：8人按权重投票，17.0赞成通过
6. **真实交付物**：FTO报告含侵权比对表/规避方案/合规留痕，可直接用于上市决策
